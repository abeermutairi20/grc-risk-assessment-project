# GRC Risk Assessment and Control Mapping Project

## Overview

This project demonstrates a practical implementation of Governance, Risk, and Compliance (GRC) by conducting a cybersecurity risk assessment and mapping identified risks to appropriate security controls and policies.

The project aligns with international standards and frameworks such as ISO 27001 and Saudi National Cybersecurity Authority (NCA) Essential Cybersecurity Controls (ECC).

---

## Objectives

* Identify cybersecurity risks across different domains
* Assess risks based on likelihood and impact
* Map risks to appropriate security controls
* Define relevant organizational policies
* Enhance security posture and support compliance readiness

---

## Scope

This project covers:

* Technical risks
* Human-related risks
* Physical security risks

---

## Risk Assessment Summary

| Risk            | Description                                                           | Likelihood | Impact | Risk Level |
| --------------- | --------------------------------------------------------------------- | ---------- | ------ | ---------- |
| Phishing Attack | Users may click malicious links leading to compromise                 | High       | High   | Critical   |
| Weak Password   | Weak passwords increase risk of unauthorized access                   | High       | High   | Critical   |
| No MFA          | Lack of multi-factor authentication increases account compromise risk | High       | High   | Critical   |
| No Backup       | Data loss or ransomware impact due to lack of backups                 | Medium     | High   | High       |
| Tailgating      | Unauthorized physical access to restricted areas                      | Medium     | Medium | Medium     |

---

## Control Mapping

| Risk            | Control                                                            | Policy                     |
| --------------- | ------------------------------------------------------------------ | -------------------------- |
| Phishing Attack | Conduct security awareness training and implement SPF, DKIM, DMARC | Email Security Policy      |
| Weak Password   | Enforce password complexity requirements                           | Password Policy            |
| No MFA          | Implement Multi-Factor Authentication (MFA)                        | Access Control Policy      |
| No Backup       | Implement automated backups and monitor backup status regularly    | Backup and Recovery Policy |
| Tailgating      | Implement access badge system and visitor management process       | Physical Security Policy   |

---

## Key Controls Implemented

* Multi-Factor Authentication (MFA)
* Security Awareness Training
* Email Security Controls (SPF, DKIM, DMARC)
* Backup and Recovery Processes
* Physical Access Controls

---

## Framework Alignment

* ISO 27001 (Access Control, Backup, Awareness, Physical Security)
* NCA ECC (Access Management, Backup, Awareness, Physical Security)

---

## Conclusion

This project demonstrates how organizations can identify cybersecurity risks and implement appropriate controls and policies to reduce risk and ensure compliance with security standards.
